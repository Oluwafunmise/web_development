Web Devlopment Project Overview

Using HTML and CSS, the project's goal was to create a website that would be both mobile-friendly and functional. The website's goal was to present a portfolio of different projects.

Approach

The principal tactic involved utilising responsive design principles. To adapt the layout, font sizes, and spacing according to the screen size, CSS media queries were used. Through the definition of breakpoints where the layout shifts, the website could fluidly adjust to varying screen sizes.

Viewport Meta Tag: It was imperative that the viewport meta tag be used in the HTML head section. By ensuring that the page is rendered by the browser at the proper width for the device, this tag helps to avoid problems like having to zoom out to view the complete page on the screen.

Optimized Images: Optimizing images for the web, including resizing and them, helped improve page loading times on mobile devices. Additionally, utilizing the max width: 100%; (95% in my own case) CSS property ensured that images scaled appropriately to fit smaller screens without losing clarity.

Challenges and Solution

Navigation: It was difficult to make the menus more manageable on smaller screens. Utilising strategies like foldable hamburger menus allowed for space conservation without sacrificing usefulness. Touch Targets: It was crucial to make sure that clickable components, like buttons and links, were proportionally sized for touch input on mobile devices. The user experience was enhanced by making these items larger. In summary, adopting responsive design concepts, content optimisation, and guaranteeing a consistent user experience across devices are all part of developing a website mobile-friendly with HTML and CSS. By implementing these tactics, the website would interact and reach visitors on mobile devices more efficiently, improving accessibility and usability.
